# Changelog

## 2026-07-20

- fix(site): `/fr/*` no longer serves duplicate content — every page answered on two URLs with identical English HTML and no canonical. `pageMeta()` now canonicals French URLs to the English original until `TRANSLATIONS_LIVE` flips, at which point each self-canonicals and gains `hreflang` pairs.
- fix(infra): the reap lifecycle rule now expires the NONCURRENT version too. The vault is versioned, so `expiration` alone only wrote a delete marker — the key left the listing (freeing the user's quota) while the bytes stayed noncurrent and billed for ever, which is the exact churn hole the 180-day alignment exists to close. A companion rule clears the delete markers.
- fix(daemon): the quota credit now retires and no longer double-counts. It was subtracted for ever, so once S3 swept the object those bytes came off usage twice and the error compounded with every delete; and a re-deposited file kept being credited as reclaimed while it was being stored again.
- fix(daemon): `Journal` transactions roll back. Three hand-rolled BEGIN/COMMIT pairs had no rollback, so one failed statement left the shared connection inside an open transaction and every later write silently vanished at exit.
- fix(daemon): blob membership records its SEAL ORDER (`blob_members.ordinal`). Orphan repair re-derived order with `newestFirst`, which reads `isFavorite` — a flag the user can toggle in Photos, silently re-ordering a blob so recomputed spans pointed at the wrong bytes. Surfaced only at restore, as garbage.
- fix(ui): `deletePath` sent `alsoIgnore` as a JSON boolean over a `[String:String]` wire, so `ControlRequest` decoding failed and EVERY delete was a no-op — invisibly, because the malformed-request reply carries `id: 0`, which no client request can match. `ParamsArg` now rejects non-string params at compile time and the cast that hid it is gone.
- feat(daemon): deleting holds and pays off — a tombstone outranks a rescan (only a re-deposit revives it), the delete confirm offers "also stop backing this up", and `reclaimedCreditBytes` returns quota the moment AWS stops billing rather than when S3 stops listing.
- refactor(daemon): `blobCap` 1 GiB → 256 MiB — a blob is the smallest unit of space reclaimable, so the cap is a deletion-granularity dial. 4× finer, and 256 MiB rather than smaller because parts are 64 MiB and four go in flight at once: a cap at the part size makes every batched blob a single part and serialises the upload.
- chore(daemon): `task daemon:gate-test` gains a fourth assertion — `PutObjectTagging` lands on a cold Deep Archive object without a thaw, as the real Cognito identity. A Mac-side version of this check tested the retired `-daemon` IAM user instead, so it could neither prove nor disprove the assumption the whole reclaim design rests on.
- feat(daemon): deleting reclaims space — once every file in a blob is tombstoned the daemon tags its object `coldstorage-reap=true` and an `expire-reclaimable-blobs` lifecycle rule expires it. The daemon gets `s3:PutObjectTagging`, never `s3:DeleteObject`.
- fix(daemon): a deposit costs what the deposit costs — `UploadEngine.run` plans only files that aren't already archived, instead of re-grouping the whole scan and re-uploading verified blobs under fresh ids.
- fix(daemon): `partsUploadConcurrentlyButNeverExceedTheCap` waits on the real condition (two parts in flight) instead of a sleep long enough to probably overlap — it failed 1 run in 3 on a loaded machine while the implementation was fine.
