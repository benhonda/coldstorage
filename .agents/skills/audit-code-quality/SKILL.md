Audit your implementation against the 4 engineering pillars defined in CLAUDE.md (simple, best-practice-following, DRY, type-safe) — don't restate them here, read them fresh so this audit never drifts from the source.

Note:

- If your implementation is extensive in nature, use agents as you see fit.
- The implementation you're auditing may already have been audited and fixed. You must not assume that it has not been fixed, nor that it has been fixed.
- If the fixes you suggest are unequivocally correct, just do it.
