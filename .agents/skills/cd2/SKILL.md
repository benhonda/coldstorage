This is the reference for **`@cdv2/email`** — the client for **CD2**, Adpharm's own
transactional email service. CD2 is a Resend-style sender backed by Amazon SES: you call a
small typed SDK, it accepts the message, and SES does the delivery. The reference pages below
are pulled **verbatim** from CD2's published docs (`docs.cd2.app`).

**The setup in one breath:** `bun add @cdv2/email`, then construct the client with an **API
key** and the **base URL** `https://send.cd2.adpharm.digital`. The sender domain in `from`
must belong to your team and have sending enabled.

**Two rules that change everything:**

1. **Import from `@cdv2/email`** — not Resend, not `nodemailer`, not a raw `fetch`. The SDK is
   zero-dependency and runs in Node, Bun, and the browser (plain `fetch`).
2. **A successful `send()` means *accepted*, not *delivered*.** The call returns once CD2 has
   queued the message; final delivery state arrives **asynchronously from SES**. To confirm
   delivery, poll `client.get(emailId)` and read the status
   (`queued → sending → sent → delivered`, or `bounced` / `failed`).

**Error handling — it never throws.** Every method returns a discriminated union
`{ data, error }`: on success `error` is null and `data` carries the result; on failure
`data` is null and `error` describes what went wrong. Branch on `error` first — don't wrap
calls in `try/catch` expecting exceptions.

**`send()` shape:** `from`, `to`, and `subject` are required; provide **at least one** of
`html` / `text`; `cc`, `bcc`, `replyTo`, and `headers` are optional.
